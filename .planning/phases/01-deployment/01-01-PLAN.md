<task type="auto">
  <name>Task 2: Create custom entrypoint script</name>
  <files>var/docker/entrypoint.sh</files>
  <action>
Create `var/docker/entrypoint.sh` that orchestrates Temporal setup and Postiz startup.

The script handles two modes:
- **TEMPORAL_EMBEDDED=true**: Run Temporal auto-setup (schema migration + server start), wait for ready, then start Postiz
- **TEMPORAL_EMBEDDED unset/false**: Skip Temporal, start Postiz directly (backward-compatible for dev)

Script structure:

```bash
#!/bin/bash
set -e

# --- Temporal Embedded Mode ---
if [ "${TEMPORAL_EMBEDDED}" = "true" ]; then
  echo "[entrypoint] Temporal embedded mode enabled"

  # Defaults consumed by auto-setup.sh (can be overridden via env)
  export DB=${DB:-postgres12}
  export POSTGRES_TLS_ENABLED=${POSTGRES_TLS_ENABLED:-true}
  export POSTGRES_TLS_DISABLE_HOST_VERIFICATION=${POSTGRES_TLS_DISABLE_HOST_VERIFICATION:-true}
  export ENABLE_ES=${ENABLE_ES:-false}
  export SKIP_ADD_CUSTOM_SEARCH_ATTRIBUTES=${SKIP_ADD_CUSTOM_SEARCH_ATTRIBUTES:-true}

  # auto-setup.sh creates temporal + temporal_visibility databases,
  # applies schema migrations, then starts temporal-server (blocking).
  # We background it so we can continue to start Postiz after it's ready.
  echo "[entrypoint] Running Temporal auto-setup..."
  /etc/temporal/auto-setup.sh &

  # Wait for Temporal gRPC port (7233) — up to 120s
  echo "[entrypoint] Waiting for Temporal server on port 7233..."
  for i in $(seq 1 60); do
    if bash -c "echo > /dev/tcp/localhost/7233" 2>/dev/null; then
      echo "[entrypoint] Temporal server is ready"
      break
    fi
    if [ "$i" -eq 60 ]; then
      echo "[entrypoint] ERROR: Temporal server did not start within 120s"
      exit 1
    fi
    sleep 2
  done
fi

# --- Start Postiz ---
echo "[entrypoint] Starting nginx and Postiz..."
nginx && pnpm run pm2
```

Key design decisions:
- auto-setup.sh is backgrounded because it starts temporal-server as a foreground process at its end. We need to continue to start Postiz.
- /dev/tcp is a bash built-in TCP check — no extra tools needed.
- TEMPORAL_EMBEDDED flag lets the same image work in environments with external Temporal.
- Defaults are set in entrypoint so only deployment-specific values (host, port, credentials) need to be in the app spec.
- The final `nginx && pnpm run pm2` matches the original CMD behavior exactly. pm2 eventually calls `pm2 logs` which keeps the process alive.

If auto-setup.sh fails in production: the fallback is to manually create the temporal and temporal_visibility databases in the managed PostgreSQL, then use temporal-sql-tool directly for schema setup. But auto-setup.sh handles this automatically and idempotently.
